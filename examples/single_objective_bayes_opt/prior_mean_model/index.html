    Custom prior mean function¶

    Here we assume we have some knowledge of the ground truth function, which we can take advantage of to speed up optimization. This "prior mean" function is specified by a pytorch module. Notice how away from the training data, the GP model reverts to the prior mean.

    In [3]:
    Copied! 
    from xopt.generators.bayesian.expected_improvement import ExpectedImprovementGenerator
from xopt.generators.bayesian.options import ModelOptions
from xopt.generators.bayesian.expected_improvement import BayesianOptions

class SquaredPrior(torch.nn.Module):
    def forward(self, X):
        return 3 * torch.sin(X.squeeze(dim=-1) + 0.25)

model_options = ModelOptions(
    mean_modules={"y": SquaredPrior()},
)
generator_options = BayesianOptions(model=model_options)
generator = ExpectedImprovementGenerator(my_vocs, options=generator_options)
generator.add_data(training_data)

    Trainable custom prior mean¶

    The custom prior mean is generally considered a fixed function with no trainable parameters on its own. In some cases though, we want the prior mean to adapt to the observed data and thus allow for any specified parameters to be trained.
